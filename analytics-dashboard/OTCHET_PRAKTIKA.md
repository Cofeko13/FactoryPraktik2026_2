## 1. Введение

В рамках учебной практики было разработано веб-приложение **Analytics Dashboard** — система для автоматического учёта посещений сайта, отображения статистики в виде графиков и таблиц, а также экспорта данных в формат Excel.

Актуальность темы связана с тем, что любой современный сайт или сервис нуждается в инструментах аналитики: владельцу важно видеть, сколько пользователей заходило, откуда они пришли (поисковики, соцсети, прямые заходы) и иметь возможность выгрузить отчёт.

Приложение реализовано на фреймворке **Laravel** с использованием современного стека: шаблонизатор Blade, библиотека Livewire для интерактивного интерфейса, Chart.js для графиков, база данных MySQL/SQLite, пакет Laravel-Excel для экспорта.

## 5. Проектирование системы

### 5.1. Общая архитектура

Приложение построено по архитектуре **MVC (Model — View — Controller)**:

- **Model** — модели `Source`, `Metric` (работа с БД).
- **View** — Blade-шаблоны и Livewire-компонент (отображение).
- **Controller** — `ExportController` (экспорт файла).

Дополнительно используется слой **Services** для бизнес-логики:
- `VisitTracker` — запись визита.
- `AnalyticsService` — чтение статистики.

### 5.2. Схема работы приложения

```
Посетитель → GET / → TrackVisit → VisitTracker → Metric::incrementVisit()
                                                      ↓
                                              таблица metrics
                                                      ↓
AnalyticsService ← Livewire дашборд ← чтение из БД
        ↓
  карточки / графики / таблица / Excel
```

### 5.3. Принципы, заложенные в проект

1. **Один способ записи** — все визиты через `Metric::incrementVisit()`.
2. **Один способ чтения** — вся аналитика через `AnalyticsService`.
3. **Единые фильтры** — period и sourceId одинаковы на дашборде и в Excel.

## 6. Описание реализованного функционала

### 6.1. Дашборд (главная страница `/`)

Реализована страница аналитики, содержащая:

**Карточки показателей:**
- «Визитов за период» — сумма визитов за выбранный период.
- «Сегодня» — визиты за текущую дату.
- «В среднем в день» — среднее арифметическое за период.

**Фильтры (Livewire, без перезагрузки страницы):**
- Период: последние 7 дней / 30 дней / все данные.
- Источник: все / Google / прямой заход / соцсети / email.

**Графики (Chart.js):**
- Столбчатый график «Визиты по дням».
- Кольцевая диаграмма «По источникам».

**Таблица:**
- 15 последних записей: дата, источник, количество визитов.

**Автообновление:**
- Карточки обновляются каждые 15 секунд (`wire:poll`).

### 6.2. Учёт посещений

При каждом заходе на главную страницу (GET-запрос):

1. Middleware `TrackVisit` перехватывает запрос.
2. `VisitTracker` определяет источник трафика.
3. `Metric::incrementVisit()` увеличивает счётчик на 1.

**Определение источника:**
- По параметру URL `?utm_source=google|email|social` (для тестирования).
- По заголовку `Referer` браузера (на реальном сайте):
  - Google, Yandex → Google
  - VK, Telegram, Facebook → Соцсети
  - Gmail, Outlook → Email
  - Прямой ввод адреса → Прямой заход

### 6.3. Экспорт в Excel

Кнопка «Скачать Excel» формирует файл `.xlsx` с колонками:
- Дата
- Источник
- Визиты

Экспорт учитывает текущие фильтры (период и источник).

### 6.4. Консольные команды (для тестирования)

| Команда | Назначение |
|---------|------------|
| `php artisan analytics:add-visits 20` | Добавить 20 тестовых визитов |
| `php artisan analytics:reset-today` | Сбросить статистику за сегодня |
| `php artisan migrate:fresh --seed` | Пересоздать БД с нуля |

## 8. База данных

### 8.1. ER-описание

**Таблица `sources`** (справочник источников):

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ |
| name | VARCHAR | Название («Google», «Соцсети») |
| slug | VARCHAR | Код (`google`, `social`) |
| created_at, updated_at | TIMESTAMP | Дата создания/обновления |

**Таблица `metrics`** (статистика визитов):

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ |
| source_id | BIGINT | Внешний ключ → sources.id |
| date | DATE | Дата |
| visits | INT | Количество визитов |
| created_at, updated_at | TIMESTAMP | Дата создания/обновления |

**Связь:** один источник → много записей метрик (1:N).

**Ограничение:** уникальная пара `(source_id, date)` — одна строка на источник в день.

### 8.2. Начальные данные

В таблицу `sources` добавлены 4 записи:
- Google
- Прямой заход
- Соцсети
- Email

Таблица `metrics` заполняется автоматически при посещениях сайта.

### 8.3. Где хранится

| Среда | Расположение |
|-------|--------------|
| Mac (разработка) | `database/database.sqlite` |
| Beget (хостинг) | MySQL, импорт через `database/mysql.sql` |

## 10. Развёртывание (локально и Beget)

### 10.1. Запуск на Mac

```bash
cd ~/Projects/analytics-dashboard
php artisan serve
```

Адрес: http://127.0.0.1:8000

Используется **Laravel Herd** (PHP 8.4). AMPPS и phpMyAdmin на Mac не требуются.

### 10.2. Размещение на Beget

1. Создать MySQL-базу в панели Beget.
2. Импортировать `database/mysql.sql` через phpMyAdmin.
3. Загрузить файлы проекта по FTP.
4. Настроить `.env` (данные MySQL, APP_URL, APP_KEY).
5. Указать корень сайта на папку `public/`.
6. Выполнить `npm run build` на Mac перед заливкой.

Подробная инструкция — в файле `BEGET.md`.
